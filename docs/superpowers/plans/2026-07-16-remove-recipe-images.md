# Remove Recipe Images Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove all hotlinked fujixweekly image references from recipe data and UI, replacing the card hero slot with the fingerprint SVG and deleting the lightbox entirely.

**Architecture:** Three sequential changes to `index.html`: (1) strip image fields from the `RECIPES` data array, (2) update `makeCard()` and the My Recipes render block to use the fingerprint SVG in place of the hero image slot, (3) delete all lightbox HTML/CSS/JS.

**Tech Stack:** Vanilla JS, inline HTML/CSS in a single `index.html` file. No build step. Preview with `python3 -m http.server 8000`.

## Global Constraints

- Single file: all changes are in `index.html`
- No build step — edit and reload in browser to verify
- `images/gear/` is unaffected — do not touch gear images
- `source_url` field on every recipe must be preserved
- The fingerprint SVG is generated by the existing `fingerprint(r)` function — call it, don't reimplement it
